📋 REQUISITOS DO SISTEMA
===================================================

1. 🐍 Python 3.8 ou superior
2. 💾 8GB RAM mínimo (16GB recomendado)
3. 🎤 Microfone funcionando
4. 🔊 Alto-falantes funcionando
5. 🌐 Internet (para download de modelos; Edge-TTS requer internet)
6. 🖥️ Sistema gráfico (para o avatar) — opcional

⚙️ INSTALAÇÃO
===================================================

1. Instale o Python 3.8+ se ainda não tiver:
   - Windows: https://www.python.org/downloads/
   - macOS: `brew install python`
   - Linux: `sudo apt install python3 python3-pip`

2. Crie um ambiente virtual (recomendado):
   ```bash
   python -m venv venv

   # Windows
   venv\Scripts\activate

   # macOS/Linux
   source venv/bin/activate
   ```

3. Instale as dependências Python:
   ```bash
   pip install -r requirements.txt
   ```

4. Dependências de sistema para o Pygame (avatar — opcional):
   **macOS:**
   ```bash
   brew install sdl2 sdl2_image sdl2_mixer sdl2_ttf portaudio
   ```
   **Ubuntu/Debian:**
   ```bash
   sudo apt-get install python3-dev libsdl2-dev libsdl2-image-dev libsdl2-mixer-dev libsdl2-ttf-dev portaudio19-dev
   ```

🔧 PROVEDOR LLM (ESCOLHA UM)
===================================================

A Chica suporta dois provedores de LLM local.

Opção A — LM Studio (padrão)
-------------------------------
1. Baixe: https://lmstudio.ai/
2. Carregue um modelo (ex: bonsai-4b)
3. Habilite a API: Settings > Enable API (porta 1234)
4. Instale: `pip install openai`

Opção B — Ollama (alternativa)
-------------------------------
1. Baixe: https://ollama.ai/
2. Inicie: `ollama serve`
3. Baixe um modelo: `ollama pull qwen3:1.7b`

🎯 CONFIGURAÇÃO
===================================================

Todas as configurações estão no arquivo `config.py`:

```python
# Provedor LLM
LLM_PROVIDER = 'lm_studio'        # 'lm_studio' ou 'ollama'
LLM_MODEL = 'bonsai-4b'
LLM_TEMPERATURE = 0.7
LLM_NUM_PREDICT = 300

# Assistente
ASSISTANT_NAME = "Chica"
AVATAR_ENABLE = True               # False desabilita o avatar

# Wake words
WAKE_WORDS = ["olá chica", "ei chica", "chica", ...]

# TTS
TTS_SYSTEM = 'kokoro'             # 'kokoro', 'edge' ou 'qwen3'
TTS_VOICE = 'pf_dora'             # Kokoro: pf_dora (pt-BR)
EDGE_TTS_VOICE = 'pt-BR-ThalitaMultilingualNeural'

# Microfone
SPEECH_THRESHOLD = 0.005
SILENCE_DURATION = 1.5
INACTIVITY_TIMEOUT = 15.0
AUDIO_DEVICE = "Isolamento de Voz"

|# Whisper (STT)
WHISPER_MODEL = 'turbo'            # 'tiny', 'base', 'small', 'medium', 'large', 'turbo'
STT_BACKEND = 'auto'               # 'auto' (detecta hardware), 'whisper' (GPU), 'faster-whisper' (CPU)
```

🚀 EXECUTANDO
===================================================

1. Certifique-se que o provedor LLM está rodando:
   - LM Studio: App aberto com API habilitada
   - Ollama: `ollama serve`

2. Execute:
   ```bash
   python app.py
   ```

3. Na inicialização, escolha o TTS:
   - 1 = Kokoro-TTS (padrão, português)
   - 2 = Qwen3-TTS (voz Serena)
   - 3 = Edge-TTS (voz Thalita, requer internet)
   - 4 = Usar config.py

DETECÇÃO INTELIGENTE DE INTENÇÃO
===================================================

A Chica usa um sistema de 2 camadas para evitar abrir
aplicativos quando você está apenas perguntando sobre eles:

Camada 1 — Heurísticas (rápida, sem LLM):
  • "calendário DOS jogos" → pergunta (não abre Calendar)
  • "QUAL o melhor navegador?" → pergunta
  • "PESQUISE no navegador sobre Python" → pesquisa web
  • Frase com "?" no final → pergunta

Camada 2 — Verificação por LLM:
  • Para casos ambíguos, consulta o modelo local
    para classificar COMANDO vs PERGUNTA
  • Comandos explícitos ("abra o navegador") são
    executados direto, sem custo extra

SISTEMA TTS
===================================================

A Chica suporta 3 sistemas de síntese de voz:

  Kokoro (padrão):
    - Local, sem internet
    - Voz pf_dora (português)
    - Rápido e leve

  Edge-TTS:
    - Requer internet
    - Voz Thalita (pt-BR, neural)
    - Qualidade excelente
    - Fallback automático para Kokoro se offline

  Qwen3-TTS:
    - Local, sem internet
    - Voz Serena
    - Qualidade avançada
    - Mais pesado que Kokoro

Se o sistema escolhido falhar, o fallback automático
para Kokoro é ativado.

DETECÇÃO AUTOMÁTICA DE HARDWARE (STT)
===================================================

A Chica detecta automaticamente o melhor backend de
reconhecimento de fala baseado no hardware:

  Apple Silicon Mac (MPS disponível)
    → whisper original (PyTorch) com GPU
    → Mais rápido no Mac

  ARM SBC / CPU sem GPU
    → faster-whisper (CTranslate2) com int8
    → 4x mais rápido que whisper original em CPU

  NVIDIA GPU (CUDA)
    → faster-whisper com CUDA (se configurado)

Você pode forçar um backend específico em config.py:
  STT_BACKEND = 'whisper'         # Forçar whisper original
  STT_BACKEND = 'faster-whisper'  # Forçar faster-whisper
  STT_BACKEND = 'auto'            # Detecção automática (padrão)

🎯 DICAS DE USO
===================================================

1. Fale claramente em ambiente silencioso
2. Aguarde a resposta antes de falar de novo
3. Use "calado" para interromper a fala
4. Avatar pode ser desabilitado com AVATAR_ENABLE = False
5. Whisper 'turbo' tem melhor custo-benefício
6. STT automático: detecta hardware e escolhe o melhor backend
7. No SBC ARM, o faster-whisper é 4x mais rápido que o whisper original

🍓 RASPBERRY PI / ORANGE PI (SBC)
===================================================

Recomendações para SBCs ARM:
  • STT automático usa faster-whisper (CTranslate2, otimizado ARM64)
  • Whisper 'tiny' para RAM limitada: WHISPER_MODEL = 'tiny'
  • Orange Pi 5 pode rodar 'turbo' (CTranslate2 é eficiente em ARM64)
  • Modelos LLM pequenos: bonsai-4b 1-bit (570MB), qwen3:0.6b
  • THINKING_ENABLED = False
  • Se não tiver internet confiável: Edge-TTS não funciona, use Kokoro
